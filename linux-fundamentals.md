# Linux Fundamentals - TryHackMe Writeup

## 🧠 Overview
This writeup combines all three parts of the Linux Fundamentals module. It covers the basics of interacting with a Linux system, navigating the filesystem, managing permissions, and using essential tools commonly used in cybersecurity.

---

## 🧩 Part 1: Linux Basics

### 🔍 What I Learned
- Why Linux is widely used in cybersecurity and modern systems
- How to interact with a Linux machine through the terminal
- Basic commands for navigating the filesystem
- Searching for files using tools like `find` and `grep`
- Introduction to shell operators to improve command usage

### 🛠️ Commands & Concepts
- `ls`, `cd`, `pwd` → navigation  
- `cat` → reading files  
- `find` → locating files  
- `grep` → searching content  
- Shell operators → combining commands  

### ⚠️ Difficulty
This part was the most challenging, as it was my first real interaction with Linux.

---

## 🧩 Part 2: Intermediate Usage

### 🔍 What I Learned
- How to connect to a remote Linux machine using SSH
- Using flags and arguments to enhance command functionality
- Managing files and directories (copying, moving, deleting)
- Understanding file permissions and user access
- Introduction to running scripts and executables

### 🛠️ Commands & Concepts
- `ssh` → remote access  
- `cp`, `mv`, `rm` → file management  
- `chmod` → permissions  
- `man` → command documentation  
- Basic scripting execution  

### ⚠️ Difficulty
This part was moderately difficult but more manageable after understanding the basics from Part 1.

---

## 🧩 Part 3: Advanced Fundamentals

### 🔍 What I Learned
- Using terminal-based text editors
- Working with utilities like downloading files and hosting simple servers
- Understanding system processes
- Automating tasks using cron jobs
- Managing packages and reviewing system logs

### 🛠️ Commands & Concepts
- Text editors (e.g., `nano`)  
- Python HTTP server for file serving  
- Process management (`ps`, `top`)  
- `crontab` → task automation  
- Package management tools  
- Log files for monitoring system activity  

### ⚠️ Difficulty
This was the easiest part, as the earlier sections built a strong foundation.

---

## 💡 Key Takeaways

- Linux is a fundamental skill for cybersecurity and penetration testing  
- Command-line efficiency improves with practice  
- File permissions and user roles are critical for system security  
- Automation and logging are essential for system maintenance and monitoring
