# Linux Shells - TryHackMe Writeup

## Overview
This room introduced Linux shells and shell scripting, focusing on how shells interact with the operating system and how scripts can be used to automate tasks and perform basic security-related functions.

---

## Tools & Concepts Used
- Linux shells (e.g., Bash)  
- Shell scripting fundamentals  
- File system navigation  
- Basic automation concepts  

---

## Methodology / Approach

### Step 1: Understanding Shells
I began by learning what a shell is and how it acts as an interface between the user and the operating system. I explored different types of Linux shells and their purposes.

---

### Step 2: Shell Scripting Fundamentals
I was introduced to shell scripting, including how scripts are structured and how they can automate repetitive tasks. This included understanding basic components such as commands, variables, and execution flow.

---

### Step 3: Practical Script Analysis
I examined a simple script designed to control access based on user input. This demonstrated how scripts can enforce basic logic, such as allowing or denying access depending on credentials.

---

### Step 4: Hands-on Exercise
I completed a practical task where I:
- Searched the system to locate a hidden shell script  
- Analyzed its behavior  
- Retrieved a hidden message from within the system  

This required navigating directories and understanding how scripts interact with files.

---

## Challenges Faced
- Understanding how scripts execute step-by-step  
- Interpreting the logic behind conditional checks  
- Navigating deeper directory structures to locate hidden files  

---

## Key Takeaways
- Linux shells are essential for interacting with and controlling the system  
- Shell scripting enables automation and efficiency  
- Even simple scripts can enforce basic security logic  
- Understanding scripts is useful for both system administration and security analysis  

---

## Reflection
This room expanded my understanding of Linux beyond basic commands into automation and scripting. It showed how scripts can be used not only for convenience but also for implementing simple control mechanisms.

I am still building familiarity with scripting, but this provided a strong introduction to how shells can be leveraged in real-world scenarios.
